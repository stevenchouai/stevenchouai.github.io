AI数字分身知识管理

我把数字分身做成了一个真正可复用的系统

过去我把数字分身当成一个实验，让它读知识库、审视网站、帮我写内容。现在我把这套流程正式抽象成一个可复用的 skill repo，同时回写到本地工作流里。这篇文章讲清它到底在做什么、为什么有用，以及它最适合帮助哪一类人。

View related repo

这次不只是“让 AI 帮我干活”

过去一段时间，我一直在做一件事：把自己的知识库、博客、网站、职业判断和写作风格逐步交给 AI，让它不只是回答问题，而是站在我的视角做事。

最早这个东西更像一个实验：

  • 读我的 Obsidian wiki
  • 理解我的关注点和写作方式
  • 审视我的个人网站
  • 帮我改页面、写博客、整理内容

但我最近发现，一个真正有价值的数字分身，不能只是“会在当前对话里表现得像我”。

它必须满足三个条件：

  1. 能检索已有资产，而不是每次从零开始
  2. 能把结果写回系统，而不是只留在聊天记录里
  3. 能从任务里沉淀规则，让后续动作越来越稳

于是我把这件事往前推了一步：把它做成了一个单独的 repo，也把它同步回我真实在用的本地 skill。

这次的目标不是展示一个酷炫 demo，而是把一个“像我”的 AI，变成一个“能长期替我工作的 operating layer”。

数字分身到底在做什么

我给这个系统的定位不是 chatbot，而是一个文件驱动的 agent layer。

它主要做四类事：

1. 读取我的长期知识

它会优先读取：

  • AGENTS.md
  • wiki/_index.md
  • wiki/outputs/
  • 相关历史文章、研究报告、复盘记录

这一步很重要。很多所谓“数字分身”其实只有一层 prompt 伪装，当前会话里看起来像你，过了这个窗口就什么都不记得了。

我想要的不是这种效果。

我想要的是：它下次再做类似任务时，知道我之前怎么想、怎么做、踩过什么坑。

2. 路由不同类型的工作

现在这个数字分身不是一个大而空的 prompt，而是明确分了几个能力模块：

  • 内容创作
  • wiki 管理
  • 代码研究
  • 网站优化
  • 简历工坊
  • 学习闭环

这意味着它不是“想到什么做什么”，而是先判断问题属于哪类，再加载对应能力文件和工作规则。

3. 把高价值输出写回系统

这是我最在意的一点。

如果一个 AI 帮我想出了一个好观点，但这个观点只出现在聊天窗口，那它几乎等于没有存在过。

所以我现在要求数字分身在完成任务后，优先把结果写回到这些地方：

  • wiki/outputs/
  • wiki/summaries/
  • Blog/Published/
  • 或项目本身的长期文档

换句话说，它要留下“资产”，而不是留下“印象”。

4. 通过 learning loop 逐步变强

真正让我开始认真看待这个系统的，是 learning loop 这一层。

它不是训练模型权重，也不是幻想 AI 自我觉醒。

它做的事情很朴素：

  • 这次任务暴露了什么稳定偏好？
  • 哪个流程应该写成默认规则？
  • 哪个错误下次应该直接跳过？
  • 哪个结果值得变成模板？

一旦这些东西被写进 durable notes，数字分身就不再只是“回答得像我”，而是开始“工作方式也越来越像我”。

这次我具体做了什么

今天我把它正式整理成了一个独立项目：

stevenchouai/digital-twin

这个 repo 不是一个复杂框架，而是一个非常克制的 skill package，核心文件包括：

  • SKILL.md：总控协议
  • WORKFLOW.md：对外解释这套 operating model
  • capabilities/：能力模块
  • references/：写作风格和标签体系
  • examples.md：典型使用方式

这次我借鉴了一些成熟 agent 系统最值得保留的原则，但没有照搬它们的整套 runtime。

我保留的是这条主链路：

retrieve -> act -> write back -> learn

也就是说：

  • 先检索已有资产
  • 再执行任务
  • 再把结果写回知识系统
  • 最后抽取下次可复用的规则

这比“我写了一个很长的 prompt”更有价值，因为它开始具备了可持续迭代的形状。

哪些人会真正需要这种东西

我不觉得每个人都需要数字分身。

如果你没有长期积累的知识资产，也没有稳定的输出需求，那它大概率只是一个很酷的概念。

但下面这几类人，我认为会明显受益。

1. 持续写作的人

如果你长期写博客、做公开表达、整理职业判断，数字分身可以帮你：

  • 从零散笔记里找主线
  • 先查历史观点，避免重复表达
  • 保持风格一致
  • 把一次输出接回整个知识系统

对这类人来说，它不是代写工具，而是一个长期共笔系统。

2. 有个人知识库的人

很多人做知识管理，停留在“我存了很多信息”。

但信息本身不会自动变成生产力。数字分身的价值在于，它把知识库从被动存档变成主动工作记忆。

你不再只是“记住了”，而是让 AI 可以基于这些知识去行动、去审视、去生成新结果。

3. 正在做个人品牌或个人网站的人

我这次最直观的应用场景，就是我的个人网站。

数字分身现在已经能做这些事：

  • 审视首页表达是否足够像我
  • 判断某个项目值不值得挂到网站上
  • 根据新主题起草一篇博客
  • 持续把网站当成一个“活系统”来维护，而不是静态简历页

如果你的网站本身就是你的长期资产，这种能力会非常有意义。

4. 想把 AI 从“对话助手”升级成“工作系统”的人

很多人现在对 AI 的使用方式还是：

“问一个问题，拿一个回答，结束。”

但我越来越确定，这不是 AI 最有杠杆的用法。

更强的用法是：

  • 把你的工作流文件化
  • 把规则显式化
  • 把输出资产化
  • 让 AI 逐步进入这个闭环

数字分身的价值，就在于它处在这个演化方向上。

它不适合什么人

这个系统也有明确边界。

它不适合：

  • 只是想要聊天陪伴的人
  • 没有知识沉淀习惯的人
  • 希望“一键自动化 everything”的人

它更像一个需要长期喂养的 operating layer，而不是开箱即用的万能 agent。

如果你没有 wiki、没有笔记、没有持续输出需求，它的收益会很有限。

我对这件事真正感兴趣的地方

我感兴趣的不是“AI 像不像我说话”。

我感兴趣的是另一件更本质的事：

一个人多年的思考、判断、偏好和方法，能不能被蒸馏成一套持续工作的系统。

如果可以，那个人知识管理的意义就变了。

我们做笔记，不再只是为了自己回头看。 我们整理观点，不再只是为了某一篇文章。 我们维护 wiki，不再只是为了显得有体系。

这些东西开始成为一个 agent 的工作记忆、操作边界和演化素材。

我觉得这才是“数字分身”真正值得做的地方。

下一步

接下来我会继续让这个数字分身做两件更实际的事：

  1. 持续维护我的个人网站和博客
  2. 继续把它的工作方法回写到本地 skill，而不是只停留在公开 repo

因为对我来说，公开展示只是副产品。

真正重要的是：它能不能在我的真实工作流里长期变强，长期有用。

如果你也在做个人知识系统、持续写作或者个人网站，我觉得这条路很值得试一试。

不是因为它酷，而是因为它可能真的会成为一种新的个人 operating system。
